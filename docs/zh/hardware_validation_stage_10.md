# Stage 10 硬件验证：机器人轨迹回放

本文适用于 `mp-robot-replay` 和 Web **真机轨迹回放**面板。它不同于离线数据查看和
策略推理。绝不可同时执行 Piper 与 RM2 门禁；任一门禁失败会阻止该机器人后续门禁。

## 前置条件

- 工作区清空，操作员掌握急停，第二观察员可看到机器人。
- 确认数据集完成、属于所选机器人，并已审查动作/状态名称、单位、语义和动作模式。
- 命令回放要求 `mp_real.replay.action_source` 和
  `mp_real.replay.action_mode=joint_position_target`；不可用原始 chunk 或模型遥测替代。
- 验证厂商部署提供停止、健康、工作区和关节限位校验。默认 Piper/RM2 适配器在这些
  校验配置前会有意拒绝执行。
- 初始速度比例为 `0.1`，永不超过 `1.0`。

## Piper 门禁

| 门禁 | 操作 | 通过条件 |
| --- | --- | --- |
| 0 | 不带 `--execute` 运行 `mp-robot-replay`。 | 得到有效计划/报告；不创建 Robot。 |
| 1 | 从 Web 连接，不确认开始。 | 可读当前状态；不运动、不连策略/相机。 |
| 2 | 以 0.1 确认移动到起点。 | 经验证的低速姿态，急停可用。 |
| 3 | 仅以 0.1 回放 3–5 个样本。 | 跟踪误差和控制时间在计划限值内。 |
| 4 | 以 0.1–0.2 回放 10–20 样本。 | 轨迹稳定且回放记录完整。 |
| 5 | 完整空载回放后停止。 | 状态进入 `ABORTED`，释放租约并记录终止。 |
| 6 | 完整空载回放后暂停/恢复。 | 任一过大姿态漂移会拒绝恢复。 |
| 7 | 已知安全对象轨迹，比例 ≤1.0。 | 已审查结果和回放记录。 |

RM2 必须使用自己的 SDK、停止/健康/限位检查和工作区清理，独立重复 0–7。Piper 通过
不能证明 RM2 的关节顺序、单位、夹爪语义或停止时延正确。

预期运动仅为以显示速度比例执行已审查 `ReplayPlan`。若发现异常移动、跟踪/健康错误、
通信丢失或操作员担忧，使用 Web 急停或厂商物理停止。worker 进入终态后断开连接，关闭
机器人并释放 `ROBOT_CONTROL`；只有控制器确认测量状态符合录制暂停状态时才能恢复。
