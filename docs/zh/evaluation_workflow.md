# 评测工作流

以下六个工作流必须明确区分：

- **数据查看**：只读录制文件，不创建 Robot、相机或 PolicyClient。Web 的 `DATA_VIEW`
  模式会将它嵌入“运行”页；只有显式提交开环任务时后台 worker 才连接 Policy，且仍绝不
  创建 Robot 或发送动作。
- **移动到数据状态**：读取 `observation.state`，预检姿态计划；真机移动需独立
  硬件安全门禁。
- **从数据状态开始部署**：把已验证的姿态连接交给新相机和策略预热；绝不将录制
  帧当作实时动作。
- **机器人轨迹回放**：对已审查动作计划进行无策略回放。
- **开环评测**：在录制观测上进行 teacher-forced 策略推理，绝不执行动作。
- **真机评测**：使用 `EvaluationSession`、人工标签和共享运行时；Baseline 使多轮
  试验可复现。

每个正式 Baseline 都应记录选用 ID、完成轮数、数据集路径、操作员、异常和中止。
建议先进行 3–5 轮冒烟，10 轮初筛，每种配置至少 20–30 轮比较，50 轮以上再形成
较稳定结论。
