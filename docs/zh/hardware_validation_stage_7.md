# Stage 7 硬件验证：Web 评测工作流

本清单供自动化假资源测试通过后、已获批准的真机验证操作员使用；它不会自动授权或
移动任何机器人。

## 测试前记录

- Git commit、机器人（Piper/RM2）、策略 URL 与标签、数据集根目录、操作员。
- 运行时配置快照、ActionSpec 与相机角色。

## 前置条件

1. 清理工作区并确认物理急停流程。
2. 确认当前速度、安全限值和复位姿态适合任务；不得为了验证提高它们。
3. 打开 Web UI，选择 `DEPLOYMENT`，连接资源并验证相机预览；不要启动普通部署控制。
4. 创建计划 3–5 轮的评测，记录生成的 session ID 与数据集路径。

## 每台机器人分别执行

| 检查 | 预期证据 |
| --- | --- |
| 无运动连接 | 出现相机和策略元数据；动作循环未运行。 |
| 预热 | 状态到达 `WAITING_RESET`；不发生机器人动作，也不创建正式 episode。 |
| 人工停止 episode | `RUNNING` → `STOPPING` → `WAITING_RESULT`；启用标签前控制线程已退出。 |
| 超时 episode | 显示 timeout 触发原因，再由操作员判为失败或无效。 |
| 无效 episode | 标记 `INVALID`；成功率分母不增加。 |
| 正常任务 episode | 标记成功或带原因的失败；写入数据集 episode 与元数据。 |
| 刷新页面 | 后端恢复状态、当前数据集路径、合法操作和标签控件。 |
| 双击 | 重复开始/标注返回清晰冲突，不创建额外动作循环或标签。 |
| 短暂网络中断 | 浏览器通过轮询恢复；后端仍是权威状态。 |
| 完成 | 数据集完成，存在 `episode_labels.jsonl`，所有评测 worker 已退出。 |

每轮后记录 episode 序号、session/generation ID、结果、失败原因、停止触发、录制器
队列/高水位/丢弃数、UI 的控制 FPS/周期/策略时延，以及原始异常类型/消息和日志。

若意外运动、安全停止、策略或录制器失败，先走物理停止流程，再在 Web 使用
**中止评测**，等待控制器与录制 worker 退出，保留 `.inprogress` 和日志；未审查
故障前不得重试同一运动。
