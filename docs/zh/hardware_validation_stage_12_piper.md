# Stage 12 Piper Baseline 验证

本清单不是移动机器人的授权。每一条可能运动的命令都必须另行获得明确授权，并先
完成 Stage 9/10 门禁。

1. 使用黑相机且不启动动作，创建 Baseline；核查 Git commit、策略标签、
   checkpoint hash（或 `null`）、ActionSpec、相机/RTC/预热与起始位姿字段。
2. 只改一个参数克隆它；核查分类 diff 和派生关系。
3. 从未变化的 Baseline 创建评测；确认仅进入 `PREPARING`，不预热、不复位、不发
   动作。
4. 获得运动授权后，做 3–5 轮空载冒烟；确认每个终态会话都附加紧凑结果和数据集
   引用。
5. 独立比较 RTC 开/关或其它单一主变量。不要把 1/1 当成稳定结论；正式比较每种
   配置使用 20–30 轮，稳定结论使用 50 轮以上。

若发生意外运动、跟踪/健康/策略/通信错误，应使用厂商物理急停或 Web 急停。恢复
前记录 Baseline ID、操作员、中止情况和数据集路径。
